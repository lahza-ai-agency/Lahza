# تحديثات Lahza — إصلاح المشاكل المذكورة

## 1) "Read case study" مش بيوصل لتفاصيل الشركة
السبب: التوجيه (routing) كان فيه باج — صفحة `/case-studies/$slug` كانت معرّفة
كـ "صفحة فرعية" تحت صفحة القايمة `/case-studies`، لكن صفحة القايمة معندهاش
`<Outlet />` لعرض الصفحة الفرعية. نتيجة كده: الرابط يتغيّر لكن المحتوى يفضل
هو نفسه (قايمة الكيس ستادييز).

**الحل:** فصلت محتوى القايمة في `case-studies.index.tsx` وسيبت
`case-studies.tsx` كـ layout خفيف بـ `<Outlet />` بس — بالظبط زي باقي الصفحات
في المشروع. دلوقتي الضغط على "Read case study" يفتح صفحة الشركة كاملة
(challenge / solution / metrics / results / gallery).

## 2) فتح بروجيكت (Open board) ومفيش تفاصيل
نفس الباج بالظبط، بس في `/projects` و `/projects/$projectId`. تم نفس الحل:
`projects.index.tsx` (القايمة) + `projects.tsx` (layout بـ Outlet) — دلوقتي
فتح أي بروجيكت يوديك على البورد بتاعه (Kanban + Project Settings) عادي.

## 3) الـ CRM ما بيظهرش عملاء + قائمة العميل في البروجيكت فاضية غير "Unassigned"
السبب الحقيقي: جدول `clients` في قاعدة البيانات منفصل عن جدول الحسابات
(`user_roles`/`profiles`). أي يوزر يسجل كـ CLIENT (سواء سجل لوحده، أو الأدمن
عمله أكونت من صفحة Admin Users) كان يتسجل دوره كـ CLIENT بس من غير ما يتعمله
صف في جدول `clients` — وده اللي خلى شاشة "Clients" في الـ CRM، وقائمة العميل
في البروجيكت، فاضيين دايمًا حتى لو فيه عملاء مسجّلين فعليًا.

**الحل (migration جديدة):** `supabase/migrations/20260621000000_auto_create_client_record.sql`
- تريجر تلقائي: أي يوزر يتحدد دوره CLIENT يتعمله صف في `clients` فورًا.
- Backfill: بيرجع يعمل صف لأي عميل حالي ناقصه الصف ده.
- كمان أضفت زرار "+ Client" في تبويب Clients بالـ CRM، عشان تقدر تسجّل عميل
  (شركة) يدويًا من غير ما يكون له حساب دخول أصلاً.
- استبدلت الـ dropdown بتاع اختيار العميل (في إنشاء/تعديل البروجيكت) بقائمة
  قابلة للبحث (اكتب اسم الشركة/العميل وهيفلتر فورًا) بدل الـ select العادي.

⚠️ **مهم جدًا:** الكود اتفحص وبني محليًا وعدّى من غير أي أخطاء، لكن لازم
**تطبّق الميجريشن الجديدة على قاعدة بيانات Supabase الحقيقية** عشان الإصلاح
يشتغل فعليًا:
```bash
supabase db push
```
(أو من Supabase Dashboard → SQL Editor، نفّذ محتوى الملف
`20260621000000_auto_create_client_record.sql` مباشرة).

## 4) مفيش Change Password في Settings
أضفت قسم "Change Password" كامل في صفحة Settings (كلمة سر جديدة + تأكيد،
تحقق من تطابقهم وطولهم، وتحديث فعلي عبر Supabase Auth).

## ملاحظة عامة
كل الإصلاحات دي على مستوى الكود وبُنيت وفُحصت (`tsc --noEmit` + `vite build`)
من غير أخطاء. لو فيه مشاكل لسه موجودة بعد رفع الميجريشن، يكون السبب غالبًا
دور (role) اليوزر بتاعك — أول حساب يتسجل في النظام بيتحول أوتوماتيك
SUPER_ADMIN، وأي حساب بعده بيتحول CLIENT بشكل افتراضي. لو محتاج تتأكد إن
حسابك أدمن، روح Settings آخر السطر هتلاقي "Role:" مكتوب جنبه دورك الحالي.
